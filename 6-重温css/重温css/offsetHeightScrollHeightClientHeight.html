# 关于元素属性offsetHeight、ScrollHeight、clientHeight

相关链接：https://juejin.cn/post/6844904110198636557

有关属性：

  • offsetHeight / offsetWidth / offsetTop
  • clientWidth / clientHeight
  • scrollWidth / scrollHeight / scrollTop

# offsetXXX (border + padding + content + 滚动条）

offsetHeight / offsetWidth 指的是一个元素的CSS【标准高宽】， 包含了边框、内边距、元素内容以及滚动条（如果存在且渲染的话），不包含 :before 或 :after 等伪类元素的高度。

offsetXXX其实就是元素在页面中的实际占用空间，同时当元素的形状发生变化时，offsetWidth就表示布局的宽，并不代表实际渲染出来的宽高，如需获取选的宽高清使用 getBoundingClientRect() 方法获取到真正的offsetWidth

# clientXXX （ padding + content）

clientHeight / clientWidth 就表示一个元素的【内容高宽】，这个属性是只读属性，对于没有定义CSS或者内联布局盒子的元素为0，否则，它是元素内部的高度(单位像素)，包含内边距，但不包括水平滚动条、边框和外边距。

clientHeight 可以通过 height + padding - 水平滚动条高度（如果存在的话）

# scrollXXX （content）

scrollWidth / scrollHeight 表示一个元素内容的实际大小，包括不在页面中的可滚动内容（内容和内边距），他是clientXXX的增强版，是整个内容大小，scrollHeight包括 ::before 和 ::after 伪元素

# xxxTop

  • offsetTop是当前元素顶部距离最近父元素顶部的距离，与有没有滚动条没有关系

  • scrollTop可以获取一个元素在垂直滚动的像素，在有滚动条的情况下，为元素可视区域距离元素顶部的像素，也就是已经滚动了多少距离

  • clientTop 一个元素顶部边框的宽度（不包括顶部外边距或内边距）

# 总结：

  • offsetXXX 是 clientXXX + 外边框 + 滚动条（有的话）
  • scrollXXX 是 clientXXX + 隐藏区域大小（如果有滚动的话）

# innerHeight innerWidth

只读属性，视口的尺寸，包括滚动条

# getBoundingClientRect()

只读，返回浮点值，常用于确定元素相对于视口的位置，该方法返回一个DOMRect对象，包含left right top bottom width height六个属性，其中

left right top bottom 都是元素相对于视口原点的距离

height width 元素的尺寸，包括被滚动隐藏的部分，padding和border参与计算，height = bottom - top

width = right - left
